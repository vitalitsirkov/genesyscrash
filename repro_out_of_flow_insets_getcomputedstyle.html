Crash anatomy

CRASH SITE (layout_box.cc:4385):
  const BoxStrut& LayoutBox::OutOfFlowInsetsForGetComputedStyle() const {
    const auto& layout_results = GetLayoutResults();
    CHECK(!layout_results.empty());   // ← fires when the LayoutBox was never laid out
    return layout_results.front()->OutOfFlowInsetsForGetComputedStyle();
  }

REQUIRED CONDITIONS:
  1. box->IsOutOfFlowPositioned() == true   (position:absolute / position:fixed / OOF-in-multicol)
  2. offset.IsFixed() == false              (top/left/right/bottom is: auto, %, calc, not a plain px value)
  3. box->GetLayoutResults().empty() == true (the LayoutBox was created but never given a layout result)

PATCHED GUARD (computed_style_utils.cc:811):
  // Before:  if (box && box->IsOutOfFlowPositioned())
  // After:   if (box && box->IsOutOfFlowPositioned() && !box->GetLayoutResults().empty())

HOW layout_results_ CAN BE EMPTY:
  a) UpdateStyleForbiddenScope is held when getComputedStyle() triggers UpdateStyleAndLayout().
     The scope makes UpdateStyleAndLayoutTree() and UpdateStyleAndLayout() return early,
     so layout never runs and layout_results_ is never populated.
     CEF 6367 enters this scope in:
       • LocalFrameView::UpdateLifecyclePhases  (via vrm_composite_tick.patch)
       • LayoutTreeScheduler::OnCompositionStarted  (layout_tree_scheduler.cc:187)
       • BlinkHelper::GetElementPositionFromViewPortById  (blink_helper.cc:203)
       • VRMHelper::GetElementLeftAndTop  (vrm_helper.cc:36)
       • (and several other blink_helper.cc functions)
  b) An OOF element is inside a CSS Grid/Table cell that runs layout passes with
     DisableLayoutSideEffectsScope active (intrinsic-size passes). In release builds
     AppendLayoutResult() is called but its DCHECK is not enforced; in debug builds
     the DCHECK fires instead.
