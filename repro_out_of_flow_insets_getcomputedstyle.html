Repro: out-of-flow getComputedStyle insets (Blink)

This page exercises ValueForPositionOffset for out-of-flow boxes using inset values that require reading layout results (LayoutBox::OutOfFlowInsetsForGetComputedStyle in an unpatched build). A crash or CHECK failure on empty GetLayoutResults() is timing- and state-dependent; it may not trigger every time. Use the unpatched binary and check the console.

If your build already guards with !GetLayoutResults().empty(), this page should only log results, not crash.

Why these cases

  • position: absolute|fixed with non-trivial insets (e.g. left: 10%, or left: 10px with anchor / position-fallback) goes through the OOF used-inset path, not the early “fixed length, no position fallback” shortcut.
  • A fragment-less LayoutBox (no entries in GetLayoutResults()) used to make that path hit CHECK / .front() on an empty vector.
